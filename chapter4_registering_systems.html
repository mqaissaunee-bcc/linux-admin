Why This Chapter Matters

System registration is one of the first tasks you'll perform when deploying a new RHEL system. Without proper registration, your system cannot receive software updates, security patches, or access Red Hat's support resources. Understanding how to register and manage subscriptions is fundamental to maintaining secure, up-to-date enterprise Linux systems.

Simple Content Access (SCA)

In 2024, Red Hat transitioned to a subscription model named Simple Content Access (SCA). This model significantly simplifies subscription management by eliminating the need to attach subscriptions on a per-system basis.

  • Organization-level validation: Subscriptions are validated at the organization level, not per-system
  • No subscription attachment: No need to manually attach subscriptions to individual systems
  • Reduced complexity: Eliminates subscription management and tooling overhead
  • Immediate content access: Systems get access to content repositories as soon as they're registered
  • Simplified inventory: Manage your entire fleet through the Hybrid Cloud Console
Aspect Traditional Model Simple Content Access
Subscription Scope Per-system attachment required Organization-wide access
Entitlement Reporting subscription-manager reports per-system Hybrid Cloud Console only
Management Portal Customer Portal (access.redhat.com) Hybrid Cloud Console (console.redhat.com)
Content Access After subscription attachment Immediately after registration
âœ… Hybrid Cloud Console

Customer-facing subscription services have moved from the Red Hat Customer Portal (access.redhat.com) to the Red Hat Hybrid Cloud Console (console.redhat.com). The Hybrid Cloud Console provides a unified experience for systems management and closer integration with Red Hat Insights.

Registering a RHEL System

You can use different options to register a system to the Red Hat subscription service. For example, you might use the graphical RHEL web console or command-line tools such as rhc (remote host configuration) or subscription-manager.

Registration Actions

You can use these tools to perform the following actions with your systems:

  • Register a system to associate it with a Red Hat account that has an active subscription. During registration, the system receives unique identity certificates for authentication.
  • Enable additional repositories to provide access to software packages as needed. Default repositories are automatically enabled for installed products.
  • Review and track available or consumed entitlements on the Subscription Inventory page of the Hybrid Cloud Console.
  • Optionally, register with Red Hat Insights for predictive analytics and proactive issue resolution.

The rhc Command (Recommended)

The rhc command-line tool is available for registering RHEL 8.8 and later systems. By default, the rhc tool enables three features: content, analytics, and remote-management. These features allow your system to access content repositories and enable Red Hat Insights analytics and remediation services with a single command.

Terminal # Register with username and password (interactive) root@host:~# rhc connect Connecting host.example.com to Red Hat. Username: your-rh-username Password: ******** Successfully connected to Red Hat! # Register with an activation key (non-interactive) root@host:~# rhc connect --activation-key=mykey --organization=12345678 # Check connection status root@host:~# rhc status Connection status: Connected to Red Hat # Disconnect a system root@host:~# rhc disconnect

The subscription-manager Command

The subscription-manager command is a traditional tool for system registration. Unlike rhc, it cannot connect your system to Red Hat Insights. After registration, you can use the insights-client command to enable Red Hat Insights.

Terminal # Register with username and password root@host:~# subscription-manager register Registering to: subscription.rhsm.redhat.com:443/subscription Username: your-rh-username Password: ******** The system has been registered with ID: abc123-def456-ghi789 # Register with activation key root@host:~# subscription-manager register --activationkey=mykey --org=12345678 # Check registration status root@host:~# subscription-manager status +-------------------------------------------+ System Status Details +-------------------------------------------+ Overall Status: Registered # Unregister a system root@host:~# subscription-manager unregister

Using Activation Keys

Activation keys are created in the Hybrid Cloud Console and allow automated, non-interactive registration. They are ideal for:

  • Automated deployments and provisioning scripts
  • Kickstart installations
  • Configuration management tools (Ansible, Puppet, etc.)
  • Cloud instance registration
Terminal # Using rhc with activation key root@host:~# rhc connect --activation-key=production-servers --organization=12345678 # Using subscription-manager with activation key root@host:~# subscription-manager register \ --activationkey=production-servers \ --org=12345678

ðŸ’¡ Recommendation

For RHEL 8.8 and later systems, use rhc connect for the simplest registration experience. It automatically enables:

  • content: Access to Red Hat software repositories
  • analytics: Connection to Red Hat Insights for predictive analytics
  • remote-management: Ability to receive automated remediation from the Hybrid Cloud Console

Managing Repositories

After registration, you may need to enable additional repositories for specific software packages. Use the subscription-manager repos command to list and manage repositories:

Terminal # List all available repositories root@host:~# subscription-manager repos --list # List only enabled repositories root@host:~# subscription-manager repos --list-enabled # Enable a specific repository root@host:~# subscription-manager repos --enable=rhel-10-for-x86_64-appstream-rpms # Disable a repository root@host:~# subscription-manager repos --disable=rhel-10-for-x86_64-supplementary-rpms

Red Hat Insights is a predictive analytics service included with all RHEL subscriptions. It identifies and remediates threats to security, performance, availability, and stability before they become outages.

Enabling Red Hat Insights

If you registered with rhc connect, Insights is automatically enabled. For systems registered with subscription-manager, enable Insights separately:

Terminal # Register with Red Hat Insights root@host:~# insights-client --register Successfully registered host host.example.com Automatic scheduling for Insights has been enabled. Starting to collect Insights data for host.example.com Writing RHSM facts to /etc/rhsm/facts/insights-client.facts ... Uploading Insights data. Successfully uploaded report from host.example.com to account 006... View the Red Hat Insights console at https://console.redhat.com/insights/ # Check Insights client status root@host:~# insights-client --status # Manually upload data to Insights root@host:~# insights-client
ðŸ”’ Data Privacy

Red Hat Insights collects system configuration data to provide recommendations. Sensitive data like passwords, keys, and user content is filtered out before transmission. You can review what data is collected using insights-client --no-upload to generate a local report.

Lab: Register Systems for Red Hat Support

In this lab, you will practice registering a RHEL system using different methods and verify the registration status.

âš ï¸ Lab Environment

This lab requires access to a RHEL system and valid Red Hat credentials. If using the classroom environment, your instructor will provide the necessary accounts and activation keys.

Task 1: Check Current Registration Status

Before registering, verify the current status of your system.

# Check if the system is registered student@workstation:~$ sudo subscription-manager status # View system identity (if registered) student@workstation:~$ sudo subscription-manager identity

Task 2: Register Using rhc connect

Register your system using the recommended rhc tool.

# Register with rhc (interactive) student@workstation:~$ sudo rhc connect # Or use an activation key (non-interactive) student@workstation:~$ sudo rhc connect --activation-key=lab-key --organization=YOUR_ORG_ID # Verify the connection student@workstation:~$ sudo rhc status

Task 3: Verify Repository Access

Confirm that your system can access Red Hat repositories.

# List enabled repositories student@workstation:~$ sudo subscription-manager repos --list-enabled # Test package download student@workstation:~$ sudo dnf check-update

Task 4: Verify Red Hat Insights

Confirm that Red Hat Insights is enabled and collecting data.

# Check Insights client status student@workstation:~$ sudo insights-client --status # Manually collect and upload data student@workstation:~$ sudo insights-client # View the Insights dashboard # Open https://console.redhat.com/insights/ in a browser

Chapter Summary

ðŸ“ Key Takeaways

  • Red Hat subscriptions provide access to software repositories, security patches, technical support, and the Knowledgebase through the Content Delivery Network (CDN).
  • Simple Content Access (SCA) is the current subscription model that validates subscriptions at the organization level, eliminating per-system attachment requirements.
  • The Hybrid Cloud Console (console.redhat.com) is the primary interface for managing subscriptions, viewing inventory, and accessing Red Hat Insights.
  • rhc connect is the recommended command for registering RHEL 8.8+ systems, automatically enabling content access, analytics, and remote management.
  • subscription-manager remains available for all RHEL versions but requires additional steps to enable Red Hat Insights.
  • Activation keys enable non-interactive, automated registration and are ideal for scripted deployments and configuration management.
  • Red Hat Insights provides predictive analytics, vulnerability detection, compliance monitoring, and automated remediation for registered systems.
