Understanding Block Devices

In Linux, storage devices are represented as Block devices transfer data in fixed-size blocks (typically 512 bytes or 4KB). They support random access, meaning you can read or write any block without accessing others. Examples include hard drives, SSDs, and USB drives. in the /dev directory. Understanding how these devices are named helps you identify and work with storage effectively.

Device Naming Convention

RHEL uses a systematic naming convention for storage devices:

Block Device Naming Structure
/dev/ sd a 1
Device Directory Driver Type Disk Letter Partition Number

Device Type Prefixes

Prefix Device Type Examples
sd SCSI/SATA/USB storage devices /dev/sda, /dev/sdb
nvme NVMe solid-state drives /dev/nvme0n1, /dev/nvme0n1p1
vd Virtual disks (KVM/QEMU) /dev/vda, /dev/vdb
sr Optical drives (CD/DVD) /dev/sr0, /dev/sr1
loop Loop devices (disk images) /dev/loop0, /dev/loop1
🛈 NVMe Device Naming

NVMe devices use a slightly different naming scheme: /dev/nvme0n1p1 where nvme0 is the controller, n1 is the namespace (usually corresponds to a drive), and p1 is the partition number.

Examining Block Device Information

RHEL provides several commands to examine block devices and their properties. The most commonly used are lsblk and blkid.

lsblk - List Block Devices

The lsblk command displays information about block devices in a tree format, showing the relationship between disks and partitions.

lsblk Examples # Basic block device listing [student@workstation ~]$ lsblk NAME MAJ:MIN RM SIZE RO TYPE MOUNTPOINTS sda 8:0 0 50G 0 disk ├─sda1 8:1 0 1G 0 part /boot └─sda2 8:2 0 49G 0 part ├─rhel-root 253:0 0 45G 0 lvm / └─rhel-swap 253:1 0 4G 0 lvm [SWAP] sdb 8:16 1 16G 0 disk └─sdb1 8:17 1 16G 0 part /mnt/usb sr0 11:0 1 1024M 0 rom # Show file system type and UUID [student@workstation ~]$ lsblk -f NAME FSTYPE FSVER LABEL UUID MOUNTPOINTS sda ├─sda1 xfs a1b2c3d4-e5f6-7890-abcd-ef1234567890 /boot └─sda2 LVM2_member LVM2 001 f0e1d2c3-b4a5-9687-6543-21fedcba0987 ├─rhel-root xfs 11111111-2222-3333-4444-555555555555 / └─rhel-swap swap 1 66666666-7777-8888-9999-000000000000 [SWAP] sdb └─sdb1 vfat FAT32 USBDATA 12345678-ABCD /mnt/usb # Show specific columns [student@workstation ~]$ lsblk -o NAME,SIZE,FSTYPE,MOUNTPOINT,UUID

blkid - Block Device Attributes

The blkid command displays attributes of block devices, particularly useful for finding UUID (Universally Unique Identifier) is a 128-bit identifier that uniquely identifies a file system. Unlike device names which can change, UUIDs remain constant and are the preferred way to reference file systems..

blkid Examples # List all block device attributes [student@workstation ~]$ blkid /dev/sda1: UUID="a1b2c3d4-e5f6-7890-abcd-ef1234567890" TYPE="xfs" /dev/sda2: UUID="f0e1d2c3-b4a5-9687-6543-21fedcba0987" TYPE="LVM2_member" /dev/mapper/rhel-root: UUID="11111111-2222-3333-4444-555555555555" TYPE="xfs" /dev/mapper/rhel-swap: UUID="66666666-7777-8888-9999-000000000000" TYPE="swap" /dev/sdb1: UUID="12345678-ABCD" LABEL="USBDATA" TYPE="vfat" # Show information for specific device [student@workstation ~]$ blkid /dev/sdb1 /dev/sdb1: UUID="12345678-ABCD" LABEL="USBDATA" TYPE="vfat" # Run as root for complete information [root@workstation ~]# blkid -p /dev/sdb1

df - Disk Free Space

The df command shows disk space usage for mounted file systems.

df Examples # Human-readable output [student@workstation ~]$ df -h Filesystem Size Used Avail Use% Mounted on /dev/mapper/rhel-root 45G 12G 33G 27% / /dev/sda1 1.0G 220M 780M 22% /boot /dev/sdb1 16G 4.0G 12G 25% /mnt/usb tmpfs 1.9G 0 1.9G 0% /dev/shm # Show file system type [student@workstation ~]$ df -hT Filesystem Type Size Used Avail Use% Mounted on /dev/mapper/rhel-root xfs 45G 12G 33G 27% / /dev/sda1 xfs 1.0G 220M 780M 22% /boot /dev/sdb1 vfat 16G 4.0G 12G 25% /mnt/usb # Show only local file systems [student@workstation ~]$ df -hlT -x tmpfs -x devtmpfs

findmnt - Find Mount Points

The findmnt command displays all mounted file systems in a tree format.

Understanding Mount Points

A A mount point is an empty directory in the existing file system hierarchy where a new file system is attached. Once mounted, the directory displays the contents of the mounted file system. is a directory in the existing file system hierarchy where you attach another file system. When a device is mounted on a directory, the contents of that directory become the contents of the mounted file system.

Common Mount Points

📂 /mnt

Traditional location for temporarily mounting file systems. Often used for manual mounts.

📷 /media

Standard location for automatically mounted removable media like USB drives and optical discs.

🏠 /run/media/$USER

Modern RHEL automatically mounts removable media here, organized by username.

💾 Custom Mounts

Administrators can create any directory as a mount point (e.g., /data, /backup).

💡 Important Concept

The mount point directory must exist before mounting. Any existing files in the mount point directory become temporarily hidden (shadowed) when a file system is mounted there. They reappear when the file system is unmounted.

Mount Point Requirements

  • The mount point directory must exist before mounting
  • Typically, mount points should be empty directories
  • You need appropriate permissions (usually root) to mount/unmount
  • Each file system can only be mounted at one location at a time
  • Mount points create a unified file system tree from multiple devices

Mounting File Systems

The mount command attaches a file system to the directory tree. You can identify the file system by device name, label, or UUID.

Basic Mount Syntax

Mount Command Syntax # General syntax mount [OPTIONS] DEVICE MOUNT_POINT # Mount by device name [root@workstation ~]# mount /dev/sdb1 /mnt/usb # Mount by UUID (recommended for persistent identification) [root@workstation ~]# mount UUID="12345678-ABCD" /mnt/usb # Mount by label [root@workstation ~]# mount LABEL="USBDATA" /mnt/usb

Creating and Using a Mount Point

Step-by-Step Mount Process # Step 1: Identify the device [student@workstation ~]$ lsblk NAME MAJ:MIN RM SIZE RO TYPE MOUNTPOINTS sdb 8:16 1 16G 0 disk └─sdb1 8:17 1 16G 0 part # Step 2: Create the mount point directory [root@workstation ~]# mkdir /mnt/data # Step 3: Mount the file system [root@workstation ~]# mount /dev/sdb1 /mnt/data # Step 4: Verify the mount [root@workstation ~]# findmnt /mnt/data TARGET SOURCE FSTYPE OPTIONS /mnt/data /dev/sdb1 vfat rw,relatime # Step 5: Access the files [student@workstation ~]$ ls /mnt/data

Mount Options

Option Description
-o ro Mount read-only
-o rw Mount read-write (default)
-o noexec Prevent execution of binaries
-o nosuid Ignore setuid/setgid bits
-o nodev Ignore device files
-t TYPE Specify file system type (ext4, xfs, vfat)
Mount with Options # Mount read-only [root@workstation ~]# mount -o ro /dev/sdb1 /mnt/usb # Mount with multiple options [root@workstation ~]# mount -o ro,noexec,nosuid /dev/sdb1 /mnt/usb # Specify file system type [root@workstation ~]# mount -t vfat /dev/sdb1 /mnt/usb
ⓘ Using UUID vs Device Name

Device names like /dev/sdb can change when devices are added or removed. The UUID remains constant for a file system, making it the preferred method for identifying file systems in configuration files and scripts.

Unmounting File Systems

Before removing a storage device, you must Unmounting ensures all cached data is written to the device and properly closes all open files. Removing a device without unmounting can cause data loss or corruption. it to ensure all data is written and no processes are accessing the files.

Using the umount Command

umount Examples # Unmount by mount point [root@workstation ~]# umount /mnt/usb # Unmount by device name [root@workstation ~]# umount /dev/sdb1 # Verify the unmount [student@workstation ~]$ findmnt /mnt/usb (no output - not mounted)
⚠ Device is Busy Error

If you get a "device is busy" error, a process is using files on the mounted file system. You need to stop those processes or change to a different directory before unmounting.

Handling Busy Mounts

Troubleshooting Busy Mounts # Check what's using the mount point [root@workstation ~]# lsof /mnt/usb COMMAND PID USER FD TYPE DEVICE SIZE/OFF NODE NAME bash 1234 student cwd DIR 8,17 4096 2 /mnt/usb vim 1456 student 4r REG 8,17 12345 100 /mnt/usb/file.txt # Alternative: use fuser to find processes [root@workstation ~]# fuser -vm /mnt/usb USER PID ACCESS COMMAND /mnt/usb: student 1234 ..c.. bash student 1456 F.... vim # Kill processes using the mount (use with caution!) [root@workstation ~]# fuser -km /mnt/usb # Lazy unmount - detach now, cleanup when idle [root@workstation ~]# umount -l /mnt/usb
  1. Change directories: Make sure you're not inside the mount point (cd ~)
  2. Close applications: Close any programs that might have files open on the device
  3. Sync data: Run sync to flush cached writes to disk
  4. Check for users: Use lsof or fuser to identify processes
  5. Unmount gracefully: Use umount command
  6. Wait for LED: If the device has an activity light, wait for it to stop blinking

Working with Removable Media

Removable media includes USB flash drives, external hard drives, SD cards, and optical media (CDs/DVDs). Modern RHEL systems automatically detect and can automatically mount these devices.

USB Flash Drives

When you connect a USB drive, the kernel detects it and creates device files. The drive typically appears as /dev/sdb, /dev/sdc, etc., depending on how many drives are already connected.

Working with USB Drives # Monitor kernel messages when inserting USB [student@workstation ~]$ dmesg | tail -20 [12345.678901] usb 2-1: new high-speed USB device number 3 [12345.789012] usb-storage 2-1:1.0: USB Mass Storage device detected [12345.890123] scsi host4: usb-storage 2-1:1.0 [12346.901234] sd 4:0:0:0: [sdb] 31266816 512-byte logical blocks [12346.912345] sd 4:0:0:0: [sdb] Write Protect is off [12347.023456] sdb: sdb1 # Check that the device appeared [student@workstation ~]$ lsblk -f # Manually mount if not auto-mounted [root@workstation ~]# mkdir -p /mnt/usb [root@workstation ~]# mount /dev/sdb1 /mnt/usb

Optical Media (CD/DVD)

Optical drives appear as /dev/sr0, /dev/sr1, etc. They are typically mounted read-only.

Working with Optical Media # Check for optical drive [student@workstation ~]$ lsblk | grep sr sr0 11:0 1 700M 0 rom # Mount a CD/DVD [root@workstation ~]# mount /dev/sr0 /mnt/cdrom mount: /mnt/cdrom: WARNING: source write-protected, mounted read-only. # Eject the disc after unmounting [root@workstation ~]# umount /mnt/cdrom [root@workstation ~]# eject /dev/sr0

ISO files are disk image files that can be mounted using A loop device is a pseudo-device that allows a regular file to be accessed as a block device. This is commonly used to mount disk image files like ISOs..

Managing Media with udisks2

The udisks2 is a daemon that handles automatic mounting and provides a D-Bus interface for managing storage devices. It allows non-root users to mount removable media in their own namespaces. service provides automatic mounting for removable media and allows users to mount devices without root privileges. It's the backend for desktop environments like GNOME.

udisksctl Command

The udisksctl command provides a command-line interface to udisks2:

udisksctl Usage # List all block devices [student@workstation ~]$ udisksctl status MODEL REVISION SERIAL DEVICE ------------------------------------------------------------- VBOX HARDDISK 1.0 VB12345678 sda Kingston DataTraveler PMAP 001A2B3C4D5E sdb # Get detailed info about a device [student@workstation ~]$ udisksctl info -b /dev/sdb1 # Mount a device (no root required for removable media) [student@workstation ~]$ udisksctl mount -b /dev/sdb1 Mounted /dev/sdb1 at /run/media/student/USBDATA # Unmount a device [student@workstation ~]$ udisksctl unmount -b /dev/sdb1 Unmounted /dev/sdb1 # Power off a USB device (safe removal) [student@workstation ~]$ udisksctl power-off -b /dev/sdb
🔐 User Mount Location

When using udisksctl, removable media is mounted at /run/media/USERNAME/LABEL where USERNAME is your login name and LABEL is the file system label or UUID.

Automatic vs Manual Mounting

Method When to Use Mount Location
Desktop Auto-mount Desktop users with GUI /run/media/$USER/
udisksctl CLI users, no root needed /run/media/$USER/
mount command Administrators, custom mount points Any directory (typically /mnt)
/etc/fstab Permanent, automated mounts Configured location

Configuring /etc/fstab

The /etc/fstab file contains static information about file systems. It defines which file systems are mounted automatically at boot and their mount options.

fstab Entry Format

/etc/fstab Field Structure

DEVICE   MOUNT_POINT   FS_TYPE   OPTIONS   DUMP   PASS

Field Description Example
DEVICE Device identifier (UUID, LABEL, or path) UUID=abc123...
MOUNT_POINT Directory where mounted /mnt/data
FS_TYPE File system type xfs, ext4, vfat
OPTIONS Mount options defaults, ro, noexec
DUMP Backup utility flag (0 = skip) 0
PASS fsck order (0 = skip, 1 = root, 2 = other) 2
Sample /etc/fstab # Device Mount Point Type Options Dump Pass UUID=a1b2c3d4-e5f6-7890-abcd-ef1234567890 /boot xfs defaults 0 0 /dev/mapper/rhel-root / xfs defaults 0 0 /dev/mapper/rhel-swap none swap defaults 0 0 UUID=12345678-ABCD /mnt/usb vfat defaults,nofail 0 0

Common Mount Options

Option Description
defaults rw, suid, dev, exec, auto, nouser, async
nofail Don't halt boot if device is missing
noauto Don't mount automatically at boot
user Allow non-root users to mount
x-systemd.automount Mount on first access (lazy mount)
⚠ Warning: nofail for Removable Media

Always use the nofail option for removable media in fstab. Without it, the system may fail to boot if the device is not connected!

Testing fstab Changes

Validating fstab # Test mounting all fstab entries (safe test) [root@workstation ~]# mount -a # Verify the mount [root@workstation ~]# findmnt --verify # Remount a file system to apply option changes [root@workstation ~]# mount -o remount /mnt/usb

Lab: Accessing Removable Storage

In this lab, you will practice mounting and unmounting storage devices, examining device information, and configuring persistent mounts.

Use various commands to examine the storage devices on your system.

# List all block devices [student@workstation ~]$ lsblk # Show file system information [student@workstation ~]$ lsblk -f # Display block device attributes [student@workstation ~]$ blkid # Show disk space usage [student@workstation ~]$ df -hT # Find current mount points [student@workstation ~]$ findmnt

Create a mount point and mount a file system manually.

# Create a mount point directory [root@workstation ~]# mkdir -p /mnt/mydata # Identify the device to mount [root@workstation ~]# lsblk -f | grep -v "loop" # Mount the device (replace sdb1 with your device) [root@workstation ~]# mount /dev/sdb1 /mnt/mydata # Verify the mount [root@workstation ~]# findmnt /mnt/mydata # List the contents [student@workstation ~]$ ls -la /mnt/mydata

Practice safely unmounting the file system.

# Make sure you're not in the mount point [student@workstation ~]$ cd ~ # Check if anything is using the mount [root@workstation ~]# lsof /mnt/mydata # Sync data to disk [root@workstation ~]# sync # Unmount the file system [root@workstation ~]# umount /mnt/mydata # Verify it's unmounted [root@workstation ~]# findmnt /mnt/mydata # (Should show no output)

Mount and unmount using udisksctl as a regular user.

# Check udisks status [student@workstation ~]$ udisksctl status # Mount using udisksctl (no root required) [student@workstation ~]$ udisksctl mount -b /dev/sdb1 Mounted /dev/sdb1 at /run/media/student/USBDATA # Access the files [student@workstation ~]$ ls /run/media/student/USBDATA # Unmount using udisksctl [student@workstation ~]$ udisksctl unmount -b /dev/sdb1 # Safely power off the device [student@workstation ~]$ udisksctl power-off -b /dev/sdb

Create a persistent mount entry in /etc/fstab.

# Get the UUID of the device [student@workstation ~]$ blkid /dev/sdb1 /dev/sdb1: UUID="12345678-ABCD" LABEL="USBDATA" TYPE="vfat" # Create mount point [root@workstation ~]# mkdir -p /mnt/persistent # Add entry to /etc/fstab (use your actual UUID) [root@workstation ~]# echo 'UUID=12345678-ABCD /mnt/persistent vfat defaults,nofail 0 0' >> /etc/fstab # Test the fstab entry [root@workstation ~]# mount -a # Verify the mount [root@workstation ~]# findmnt /mnt/persistent
⚠ Caution

Always use nofail for removable devices in fstab to prevent boot failures when the device is not connected.

Chapter Summary

Key Takeaways

  • Block devices represent storage in /dev with naming conventions like sda (SCSI/SATA), nvme0n1 (NVMe), and sr0 (optical).
  • lsblk and blkid are essential commands for examining block device information, UUIDs, and file system types.
  • Mount points are directories where file systems are attached to the directory tree hierarchy.
  • The mount command attaches file systems; always specify device and mount point. Use UUID for reliable identification.
  • The umount command safely detaches file systems. Always unmount before removing devices.
  • udisks2 provides user-friendly mounting for removable media without root privileges.
  • /etc/fstab configures persistent mounts. Use nofail option for removable devices.
