Introduction to File Management

Effective file management is a fundamental skill for Linux system administrators. The command line provides powerful tools that enable you to create, organize, and manipulate files and directories with precision and efficiency far beyond what graphical file managers offer.

In this chapter, you'll learn the core commands that form the foundation of file management in Linux. These commands‚Äîtouch, mkdir, cp, mv, rm, and rmdir‚Äîwill become essential tools in your daily administration tasks.

Linux is case-sensitive. File.txt, file.txt, and FILE.TXT are three different files. Also, filenames beginning with a dot (.) are hidden filesHidden files don't appear in normal directory listings. Use ls -a to see them. They're typically used for configuration files in home directories. and won't appear in regular ls output.

Creating Files and Directories

Before you can manage files, you need to create them. Linux provides simple yet powerful commands for creating both files and directories.

touch - Create Files and Update Timestamps

The touch command creates empty files or updates the timestamp of existing files to the current time.

touch Examples # Create a single empty file student@workstation:~$ touch newfile.txt # Create multiple files at once student@workstation:~$ touch file1.txt file2.txt file3.txt # Verify the files were created student@workstation:~$ ls -l *.txt -rw-r--r--. 1 student student 0 Jan 15 10:30 file1.txt -rw-r--r--. 1 student student 0 Jan 15 10:30 file2.txt -rw-r--r--. 1 student student 0 Jan 15 10:30 file3.txt -rw-r--r--. 1 student student 0 Jan 15 10:30 newfile.txt # Update timestamp of existing file student@workstation:~$ touch file1.txt student@workstation:~$ ls -l file1.txt -rw-r--r--. 1 student student 0 Jan 15 10:35 file1.txt

Key touch Options

  • -a ‚Äî Change only the access time
  • -m ‚Äî Change only the modification time
  • -t [[CC]YY]MMDDhhmm[.ss] ‚Äî Set a specific timestamp
  • -c ‚Äî Do not create file if it doesn't exist

mkdir - Create Directories

The mkdir command creates new directories. You can create single directories, multiple directories, or entire directory trees.

mkdir Examples # Create a single directory student@workstation:~$ mkdir projects # Create multiple directories at once student@workstation:~$ mkdir docs images scripts # Create nested directories with -p (parent) student@workstation:~$ mkdir -p projects/web/css # This creates 'projects', 'web' inside it, and 'css' inside 'web' student@workstation:~$ ls -R projects projects: web projects/web: css # Create directory with specific permissions student@workstation:~$ mkdir -m 700 private student@workstation:~$ ls -ld private drwx------. 2 student student 4096 Jan 15 10:40 private
‚ö†Ô∏è The -p Option is Essential

Without -p, mkdir will fail if the parent directory doesn't exist. Always use mkdir -p when creating nested directory structures to avoid errors.

Copying Files and Directories

The cp command copies files and directories. Understanding its options is crucial for effective file management, especially when working with directories or preserving file attributes.

Basic cp Usage # Copy a file to a new name in the same directory student@workstation:~$ cp original.txt backup.txt # Copy a file to a different directory student@workstation:~$ cp report.txt ~/Documents/ # Copy a file to a directory with a new name student@workstation:~$ cp report.txt ~/Documents/annual_report.txt # Copy multiple files to a directory student@workstation:~$ cp file1.txt file2.txt file3.txt ~/backup/ # Copy using current directory (.) as destination student@workstation:~/projects$ cp /etc/hosts .

Copying Directories

To copy directories, you must use the -r (recursive) option:

Copying Directories # Copy a directory and all its contents student@workstation:~$ cp -r projects projects_backup # Copy directory preserving all attributes (permissions, timestamps) student@workstation:~$ cp -a projects projects_archive # Copy with verbose output to see what's being copied student@workstation:~$ cp -rv projects/ /tmp/backup/ 'projects/' -> '/tmp/backup/projects' 'projects/file1.txt' -> '/tmp/backup/projects/file1.txt' 'projects/docs' -> '/tmp/backup/projects/docs'

Important cp Options

Option Description
-r, -R Copy directories recursively (required for directories)
-a Archive mode: preserves permissions, ownership, timestamps, and links
-i Interactive: prompt before overwriting existing files
-v Verbose: show files as they are copied
-n No clobber: don't overwrite existing files
-u Update: copy only when source is newer than destination
-p Preserve mode, ownership, and timestamps
üí° Best Practice: Use -i for Safety

When copying files that might overwrite existing data, use cp -i to get a confirmation prompt. Many administrators add alias cp='cp -i' to their shell configuration for safety.

5

Moving and Renaming Files

The mv command serves two purposes: moving files to different locations and renaming files. Unlike cp, the mv command works on directories without needing the -r option.

mv Examples # Rename a file student@workstation:~$ mv oldname.txt newname.txt # Move a file to another directory student@workstation:~$ mv report.txt ~/Documents/ # Move and rename in one operation student@workstation:~$ mv draft.txt ~/Documents/final_report.txt # Move multiple files to a directory student@workstation:~$ mv file1.txt file2.txt file3.txt ~/archive/ # Move a directory (no -r needed) student@workstation:~$ mv projects/ ~/work/ # Rename a directory student@workstation:~$ mv old_project new_project # Interactive move (prompt before overwriting) student@workstation:~$ mv -i source.txt destination.txt mv: overwrite 'destination.txt'? y

Key mv Options

Option Description
-i Interactive: prompt before overwriting
-f Force: don't prompt, always overwrite
-n No clobber: don't overwrite existing files
-v Verbose: explain what is being done
-u Update: move only when source is newer
üìù Move vs Copy Behavior

When moving files within the same filesystem, mv simply renames the file entry‚Äîit's nearly instantaneous regardless of file size. When moving across filesystems, mv actually copies the data and then deletes the original, which takes longer for large files.

Deleting Files and Directories

The rm command removes files, while rmdir removes empty directories. These commands are powerful but potentially dangerous‚Äîdeleted files cannot be recovered from a recycle bin.

‚ö†Ô∏è Warning: No Undo!

Linux has no recycle bin or trash can at the command line. When you delete a file with rm, it's gone. Always double-check your commands, especially when using wildcards or the -r option.

rm - Remove Files and Directories

rm Examples # Remove a single file student@workstation:~$ rm unwanted.txt # Remove multiple files student@workstation:~$ rm file1.txt file2.txt file3.txt # Remove with confirmation prompt (safer) student@workstation:~$ rm -i important.txt rm: remove regular file 'important.txt'? n # Remove a directory and all its contents student@workstation:~$ rm -r old_project/ # Remove directory with confirmation for each file student@workstation:~$ rm -ri projects/ # Force remove without prompts (DANGEROUS!) student@workstation:~$ rm -rf temp_files/ # Remove files starting with a dash student@workstation:~$ rm -- -badfilename # OR student@workstation:~$ rm ./-badfilename

Important rm Options

Option Description
-i Interactive: prompt before every removal
-I Prompt once before removing more than 3 files
-r, -R Recursive: remove directories and their contents
-f Force: ignore nonexistent files, never prompt
-v Verbose: explain what is being done
-d Remove empty directories (like rmdir)

rmdir - Remove Empty Directories

The rmdir command only removes empty directories, making it safer than rm -r.

rmdir Examples # Remove an empty directory student@workstation:~$ rmdir empty_folder # Attempt to remove non-empty directory (will fail) student@workstation:~$ rmdir projects rmdir: failed to remove 'projects': Directory not empty # Remove nested empty directories with -p student@workstation:~$ rmdir -p a/b/c # Removes 'c', then 'b', then 'a' if each becomes empty # Remove multiple empty directories student@workstation:~$ rmdir dir1 dir2 dir3

When to Use rmdir vs rm -r

  • rmdir: Use when you only want to remove truly empty directories
  • rm -r: Use when you need to remove directories with contents
  • rm -ri: Use when you want confirmation for each file

Wildcards and Globbing

GlobbingGlobbing is filename expansion using special characters (wildcards) that the shell interprets to match multiple files. It's named after the glob() function. is the shell's pattern-matching capability that allows you to work with multiple files efficiently. The shell expands wildcard characters before the command is executed.

Practical Wildcard Examples

Globbing in Action # List all files (asterisk matches any characters) student@workstation:~$ ls * # List all .txt files student@workstation:~$ ls *.txt # List files starting with 'report' student@workstation:~$ ls report* # List files with exactly one character before .txt student@workstation:~$ ls ?.txt a.txt b.txt 1.txt # List files starting with a, b, or c student@workstation:~$ ls [abc]* # List files starting with a lowercase letter student@workstation:~$ ls [a-z]* # List files NOT starting with a number student@workstation:~$ ls [!0-9]* # Copy all .log files to archive student@workstation:~$ cp *.log ~/archive/ # Remove all backup files (ending in ~) student@workstation:~$ rm *~ # List hidden files (except . and ..) student@workstation:~$ ls .[!.]*
üîç Shell Expansion Order

The shell expands wildcards before passing arguments to commands. If you type ls *.txt and there are three .txt files, the shell runs ls file1.txt file2.txt file3.txt. If no files match, the behavior depends on the shell settings.

Hard Links and Symbolic Links

Linux provides two types of links that allow multiple filenames to reference the same data: hard linksA hard link is a directory entry that points directly to a file's inode. Multiple hard links to a file share the same data blocks and metadata. and symbolic linksA symbolic link (symlink) is a special file that contains a path to another file or directory. It's similar to a shortcut in Windows. (also called soft links or symlinks).

Understanding Links

Hard Links: Multiple directory entries point directly to the same inode. Both filenames are equal‚Äîthere is no "original" and "link."

Comparing Hard Links and Symbolic Links

Feature Hard Link Symbolic Link
Points to Inode (data directly) Pathname (another filename)
Cross filesystems No Yes
Link to directories No (usually) Yes
If original deleted Data persists Link breaks (dangling)
Created with ln source link ln -s source link
Inode number Same as original Different from original

Creating Links

ln Command Examples # Create a hard link student@workstation:~$ ln original.txt hardlink.txt # Verify both have the same inode student@workstation:~$ ls -li original.txt hardlink.txt 12345678 -rw-r--r--. 2 student student 100 Jan 15 10:00 hardlink.txt 12345678 -rw-r--r--. 2 student student 100 Jan 15 10:00 original.txt # Create a symbolic link student@workstation:~$ ln -s /etc/hosts hosts_link # Symbolic link shows as 'l' type and shows target student@workstation:~$ ls -l hosts_link lrwxrwxrwx. 1 student student 10 Jan 15 10:05 hosts_link -> /etc/hosts # Create a symbolic link to a directory student@workstation:~$ ln -s /var/log logs student@workstation:~$ ls logs/ messages secure audit ... # Find broken symbolic links student@workstation:~$ find . -xtype l

Searching for Files

Linux provides two main commands for finding files: find for real-time searches with powerful filtering, and locate for fast searches using a pre-built database.

find - Search in Real-Time

The find command searches the filesystem in real-time with numerous criteria options.

find Examples # Find files by name (case-sensitive) student@workstation:~$ find /home -name "*.txt" # Find files by name (case-insensitive) student@workstation:~$ find /home -iname "*.TXT" # Find files by type (f=file, d=directory, l=link) student@workstation:~$ find /etc -type f -name "*.conf" # Find directories only student@workstation:~$ find /var -type d -name "log*" # Find files larger than 100MB student@workstation:~$ find / -type f -size +100M 2>/dev/null # Find files modified in the last 7 days student@workstation:~$ find /var/log -type f -mtime -7 # Find files by owner student@workstation:~$ find /home -user student # Find and execute a command on results student@workstation:~$ find /tmp -name "*.tmp" -exec rm {} \; # Find with multiple conditions (AND) student@workstation:~$ find /var -type f -name "*.log" -size +1M # Find with OR condition student@workstation:~$ find . -name "*.txt" -o -name "*.md"

Common find Options

Option Description
-name Match by filename (case-sensitive)
-iname Match by filename (case-insensitive)
-type f/d/l Match by type: file, directory, or link
-size +/-n Match by size (c=bytes, k=KB, M=MB, G=GB)
-mtime +/-n Match by modification time (days)
-user Match by owner
-perm Match by permissions
-exec cmd {} \; Execute command on each result

locate - Fast Database Search

The locate command searches a pre-built database of filenames, making it extremely fast but potentially outdated.

locate Examples # Basic search (very fast) student@workstation:~$ locate passwd /etc/passwd /etc/passwd- /usr/bin/passwd /usr/share/man/man1/passwd.1.gz /usr/share/man/man5/passwd.5.gz # Case-insensitive search student@workstation:~$ locate -i README # Count matches instead of showing them student@workstation:~$ locate -c "*.conf" 1847 # Show only files that still exist student@workstation:~$ locate -e passwd # Update the locate database (requires root) root@workstation:~# updatedb # Limit number of results student@workstation:~$ locate -l 5 "*.log"
üìÖ Database Updates

The locate database is typically updated daily via a cron job running updatedb. Files created after the last database update won't be found. Run sudo updatedb to refresh the database manually.

find vs locate: When to Use Each

Use Case Recommended Command
Quick filename search locate
Find recently created files find
Search by size, permissions, or date find
Execute commands on found files find
Search specific directory tree find

Hands-on Lab

Practice the file management commands you've learned with these hands-on exercises.

Create a project directory structure with files and subdirectories.

# Create the main project directory and subdirectories student@workstation:~$ mkdir -p project/{src,docs,tests,config} # Verify the structure student@workstation:~$ ls -R project/ project/: config docs src tests # Create some files student@workstation:~$ touch project/src/main.py project/src/utils.py student@workstation:~$ touch project/docs/README.md project/docs/API.md student@workstation:~$ touch project/config/settings.conf student@workstation:~$ touch project/tests/test_main.py # View the complete structure student@workstation:~$ find project -type f project/src/main.py project/src/utils.py project/docs/README.md project/docs/API.md project/config/settings.conf project/tests/test_main.py

Practice copying and moving files within the project structure.

# Create a backup of the config directory student@workstation:~$ cp -r project/config project/config_backup # Copy a file with a new name student@workstation:~$ cp project/docs/README.md project/docs/README_old.md # Move a file to a different directory student@workstation:~$ mv project/docs/API.md project/src/ # Rename a file student@workstation:~$ mv project/src/utils.py project/src/helpers.py # Verify the changes student@workstation:~$ find project -type f | sort

Practice using wildcards to work with multiple files.

# List all .py files student@workstation:~$ ls project/*/*.py # List all .md files student@workstation:~$ ls project/docs/*.md # Copy all Python files to tests directory student@workstation:~$ cp project/src/*.py project/tests/ # Create numbered backup files student@workstation:~$ touch project/backup{1,2,3}.log # List files with numbers in the name student@workstation:~$ ls project/*[0-9]*

Create hard links and symbolic links.

# Create a hard link student@workstation:~$ ln project/config/settings.conf project/settings_link # Verify they share the same inode student@workstation:~$ ls -li project/config/settings.conf project/settings_link # Create a symbolic link to the docs directory student@workstation:~$ ln -s project/docs documentation # Verify the symbolic link student@workstation:~$ ls -l documentation lrwxrwxrwx. 1 student student 12 Jan 15 11:00 documentation -> project/docs # Access files through the link student@workstation:~$ ls documentation/

Use find and locate to search for files.

# Find all Python files in project student@workstation:~$ find project -name "*.py" # Find all files modified today student@workstation:~$ find project -type f -mtime 0 # Find empty files student@workstation:~$ find project -type f -empty # Find and list files with their details student@workstation:~$ find project -type f -exec ls -l {} \; # Use locate to find system configuration files student@workstation:~$ locate -l 10 "*.conf"

Practice safe deletion of files and directories.

# Remove backup files (interactive) student@workstation:~$ rm -i project/backup*.log # Remove empty directories student@workstation:~$ rmdir project/config_backup # Remove the symbolic link student@workstation:~$ rm documentation # Clean up entire project (be careful!) student@workstation:~$ rm -ri project/

Chapter Summary

Key Takeaways

  • touch creates empty files or updates timestamps; mkdir creates directories (use -p for nested directories).
  • cp copies files and directories (use -r for directories, -a to preserve attributes).
  • mv moves or renames files and directories without needing special options for directories.
  • rm removes files and directories (use -r for directories, -i for safety); rmdir removes only empty directories.
  • Wildcards expand filenames: * matches any characters, ? matches one character, [abc] matches specific characters.
  • Hard links point directly to inodes and persist when the original is deleted; symbolic links point to pathnames and can link across filesystems.
  • find searches in real-time with powerful filtering; locate searches a database quickly but may be outdated.
