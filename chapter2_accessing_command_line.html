Why Learn the Command Line?

The command line is the most powerful and flexible way to interact with a Linux system:

  • Efficiency: Many tasks are faster at the command line than using a GUI
  • Automation: Commands can be combined into scripts for repeatable tasks
  • Remote Access: The command line works over slow network connections
  • Servers: Most Linux servers don't have a graphical environment
  • Troubleshooting: Many diagnostic tools are command-line only

Accessing the Command Line in a Text Interface

Understanding Terminals and Consoles

A terminal is a text-based interface to enter commands into and print output from a computer system. To run the shell, you must log in to the computer on a terminal.

Physical Console

A hardware keyboard and display for input and output might be directly connected to the computer. This is the physical console from the Linux machine. The physical console supports multiple virtual consoles, which can run on separate terminals. Each virtual console supports an independent login session.

Many system administrators choose not to run a graphical environment on their servers because users do not log in to servers as a desktop workspace. A server's workload can more effectively use the significant resources that a graphical environment consumes.

Virtual Consoles in RHEL 10

You can switch between the virtual consoles by pressing Ctrl+Alt and a function key (F1 through F6) at the same time. Most of these virtual consoles run a text login prompt.

A headless server does not have a keyboard and display permanently connected to it. A data center might be filled with many racks of headless servers, and not providing each with a keyboard and display saves space and expense.

For administrators to log in, a login prompt for a headless server might be provided by its serial console, which runs on a serial port that is connected to a networked console server for remote access.

The serial console is typically used to access the server if the server network card becomes misconfigured and logging into the server over the conventional network connection becomes impossible.

Logging in to a Remote System

Linux users and administrators often need to get shell access to a remote system by connecting to it over the network. In a modern computing environment, many headless servers are virtual machines or are running as public or private cloud instances. These systems are not physical and do not have real hardware consoles.

In Linux, the most common way to get a shell prompt on a remote system is to use Secure Shell (SSH). Most Linux systems (including Red Hat Enterprise Linux) and macOS provide the OpenSSH command-line program ssh for this purpose.

SSH Connection Example
user@host:~$ ssh remoteuser@remotehost
remoteuser@remotehost's password: ********
remoteuser@remotehost:~$ _

The ssh command encrypts the connection to secure the communication against eavesdropping or hijacking of the passwords and content.

Accessing the Command Line with the Desktop Environment

The GNOME Desktop Environment

The desktop environment is the graphical user interface on a Linux system. GNOME 47 is the default desktop environment in Red Hat Enterprise Linux 10. GNOME provides an integrated desktop for users and a unified development platform on top of a graphical framework that is powered by Wayland.

The highly customizable GNOME Shell provides the core user interface functions for the desktop environment. In RHEL 10, the default GNOME Shell appearance is the standard GNOME theme. You can switch from the default to the GNOME Classic theme, which is closer to the appearance of earlier versions of the GNOME desktop.

Parts of the GNOME Shell

The GNOME Shell user interface includes several major elements. Click each element to learn more:

Top Bar

The top bar provides a consistent interface element across all workspaces. It contains:

  • The Activities button (top-left corner)
  • A clock in the center (shows date, time, and notifications)
  • System menu icons on the right (network, volume, power)

Activities Overview

The Activities button (or pressing the Super key) opens the Activities Overview, which provides:

  • A search bar to find applications, files, and settings
  • A view of all open windows
  • Access to the application grid
  • Workspace switcher on the right side

Workspaces

Workspaces are virtual desktops that help organize your open windows:

  • GNOME creates workspaces dynamically as needed
  • Drag windows between workspaces in Activities Overview
  • Use Super+Page Up/Down to switch workspaces
  • Helps keep related tasks organized and reduces clutter

Dash (Dock)

The Dash appears in the Activities Overview and provides quick access to:

  • Favorite applications (pinned by you)
  • Currently running applications (with indicator dots)
  • The application grid button (show all installed apps)
  • Right-click apps for additional options

System Menu

The System Menu (top-right corner) provides quick access to:

  • Volume control and audio settings
  • Network connections (Wi-Fi, Ethernet)
  • Screen brightness
  • Power options (suspend, restart, power off)
  • Settings and lock screen options

Executing Commands with the Bash Shell

The Shell Prompt

When you log in or open a terminal, you see the shell prompt. The default prompt format in Bash provides useful information:

Shell Prompt Format student@workstation:~$ _ # Let's break this down: # student = username (who you are logged in as) # @ = separator # workstation = hostname (the computer's name) # : = separator # ~ = current directory (~ means home directory) # $ = regular user (# would mean root/superuser)
âš ï¸ Important: The Root Prompt

When logged in as the root (superuser) account, the prompt ends with # instead of $. The root user has full system accessâ€”be careful with commands when you see the # prompt!

Basic Command Syntax

Commands in Linux follow a consistent pattern:

Command Structure

command [options] [arguments] # command = the program to run # options = modify how the command behaves (start with - or --) # arguments = what the command acts on (files, directories, etc.)

Options come in two formats:

Type Format Example Can Combine?
Short Options Single dash + letter -l, -a, -h Yes: -lah
Long Options Double dash + word --long, --all, --help No, must be separate

Many options have both short and long forms. For example, ls -a and ls --all do the same thing.

Tab Completion

Tab completion is one of the most useful features of the Bash shell. Press Tab to automatically complete:

  • Commands: Type the first few letters and press Tab
  • File and directory names: Works with paths too
  • Options: After typing --, Tab shows available options

Essential Commands

Let's explore some fundamental commands you'll use frequently. Each command is shown with examples.

date - Display Date and Time

The date command displays the current date and time. You can format the output using various options.

student@workstation:~$ date Mon Jan 15 10:30:45 EST 2026 student@workstation:~$ date +%Y-%m-%d 2026-01-15 student@workstation:~$ date +%H:%M:%S 10:30:45 student@workstation:~$ date +%s 1768595445 # Epoch time (seconds since Jan 1, 1970)

passwd - Change Password

The passwd command changes a user's password. Regular users can only change their own password.

student@workstation:~$ passwd Changing password for student. Current password: ******** New password: ******** Retype new password: ******** passwd: password updated successfully
ðŸ”’ Password Requirements

RHEL enforces password quality rules. Passwords typically must be at least 8 characters, include mixed case, numbers, and special characters, and not be based on dictionary words.

file - Determine File Type

The file command examines files and reports their type. Unlike some operating systems, Linux does not rely on file extensions.

student@workstation:~$ file /etc/passwd /etc/passwd: ASCII text student@workstation:~$ file /bin/ls /bin/ls: ELF 64-bit LSB pie executable, x86-64... student@workstation:~$ file /usr/share/icons/hicolor/48x48/apps/*.png /usr/.../app.png: PNG image data, 48 x 48, 8-bit/color RGBA... student@workstation:~$ file /dev/sda /dev/sda: block special (8/0)

cat - Concatenate and Display Files

The cat command displays file contents. It can also concatenate multiple files.

student@workstation:~$ cat /etc/passwd root:x:0:0:root:/root:/bin/bash bin:x:1:1:bin:/bin:/sbin/nologin daemon:x:2:2:daemon:/sbin:/sbin/nologin ... # Display with line numbers student@workstation:~$ cat -n /etc/passwd 1 root:x:0:0:root:/root:/bin/bash 2 bin:x:1:1:bin:/bin:/sbin/nologin ... # Concatenate multiple files student@workstation:~$ cat file1 file2 file3

head and tail - View File Beginnings and Endings

head shows the first lines of a file. tail shows the last lines. Both default to 10 lines.

# First 10 lines (default) student@workstation:~$ head /etc/passwd # First 5 lines student@workstation:~$ head -n 5 /etc/passwd root:x:0:0:root:/root:/bin/bash bin:x:1:1:bin:/bin:/sbin/nologin daemon:x:2:2:daemon:/sbin:/sbin/nologin adm:x:3:4:adm:/var/adm:/sbin/nologin lp:x:4:7:lp:/var/spool/lpd:/sbin/nologin # Last 3 lines student@workstation:~$ tail -n 3 /etc/passwd # Follow a file in real-time (great for logs!) student@workstation:~$ tail -f /var/log/messages

wc - Word Count

The wc command counts lines, words, and bytes/characters in files.

student@workstation:~$ wc /etc/passwd 45 89 2678 /etc/passwd # Lines Words Bytes Filename # Count only lines student@workstation:~$ wc -l /etc/passwd 45 /etc/passwd # Count only words student@workstation:~$ wc -w /etc/passwd 89 /etc/passwd # Count only characters student@workstation:~$ wc -c /etc/passwd 2678 /etc/passwd

Quick Reference: Essential Commands

Command Purpose Common Options

cat Display file contents -n (number lines)
head Show first lines -n NUM
tail Show last lines -n NUM, -f (follow)

Chapter Summary

Key Takeaways

  • The Bash shell is the default command-line interpreter in Red Hat Enterprise Linux, providing a text-based interface to run commands and scripts.
  • Virtual consoles (tty1-tty6) provide multiple independent login sessions on a single physical console. Use Ctrl+Alt+F1-F6 to switch between them.
  • GNOME 47 is the default desktop environment in RHEL 10, running on the Wayland display server. Start a terminal from Activities or the application grid.
  • SSH provides secure, encrypted remote access to Linux systems. Use ssh user@hostname to connect.
  • The shell prompt shows username, hostname, current directory, and user type ($ for regular user, # for root).
  • Tab completion saves typing time by auto-completing commands, filenames, and options when you press Tab.
  • Command history lets you recall and reuse previous commands with arrow keys, Ctrl+R, or the history command.
  • Essential commands include date, passwd, file, cat, head, tail, and wc for basic system interaction and file viewing.
