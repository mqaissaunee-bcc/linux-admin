The TCP/IP Network Model

The TCP/IP network model is a four-layer set of communication protocols specified by RFC 1122. It describes how data is packetized, addressed, transmitted, routed, and received between computers over a network.

Application Layer

Each application has specifications for communication. Common protocols include: SSH (port 22), HTTPS (port 443), FTP (ports 20-21), SMTP (port 25), DNS (port 53), and DHCP (ports 67-68).

Transport Layer

TCPTransmission Control Protocol. Provides reliable, connection-oriented communication ensuring data delivery and correct order. provides reliable, connection-oriented communication. UDPUser Datagram Protocol. A connectionless protocol offering faster but less reliable communication. is connectionless and faster but less reliable. An IP address plus port forms a socketA network endpoint defined by the combination of an IP address and a port number (e.g., 192.168.1.10:80).. The /etc/services file lists well-known port assignments.

Internet Layer

Responsible for transmitting data packets from source to destination across interconnected networks. Key protocols include IPv4 and IPv6. Each host has a unique IP address and a prefixThe portion of an IP address identifying the network. Expressed in CIDR notation (/24) or as a netmask (255.255.255.0). determining its network. Routers forward traffic between networks.

Link Layer

Provides connection to physical media. Common types are wired Ethernet (802.3) and wireless Wi-Fi (802.11). Each device has a MAC addressMedia Access Control address. A unique hardware address assigned to a network interface (e.g., 52:54:00:00:fa:0a). to identify it on the local segment.

TCP/IP vs. OSI Model

The OSI modelOpen Systems Interconnection model. A seven-layer reference model for network communication. remains a popular reference. You may see references to "Layer 2" (Ethernet), "Layer 3" (IP), or "Layer 4" (TCP).

IPv4 Addressing

An IPv4 addressA 32-bit number expressed as four 8-bit octets in decimal format (0-255), separated by dots. Divided into a network prefix and host identifier. is a 32-bit number expressed as four octets (0–255) separated by dots. It has two parts:

  • Network prefix — Identifies the subnet; all hosts on the same subnet share this
  • Host identifier — Identifies a specific host on the subnet

Network Masks

A netmaskA binary mask indicating how many bits belong to the network prefix. Expressed in CIDR notation (/24) or dotted-decimal (255.255.255.0). indicates how many bits belong to the network prefix:

  • CIDR notation — /24
  • Dotted-decimal — 255.255.255.0

Address Configuration

A DHCPDynamic Host Configuration Protocol. Automatically assigns IP addresses, subnet masks, gateways, and DNS servers to clients. server assigns an IPv4 address, subnet mask, default gateway, and DNS servers. The lease must be periodically renewed.

Static settings are read from local configuration files. The administrator manually specifies the IP address, netmask, gateway, and DNS servers.

IPv4 Subnetting

The number of available host addresses depends on the network prefix size. Each subnet has three special addresses:

  • Network address — Lowest address (host bits all zeros)
  • Broadcast address — Highest address (host bits all ones)
  • Gateway address — Commonly the first usable host address

Example: 192.168.1.107/24

IPv4 Routing

Each host uses a routing tableA data structure listing destination networks, the interface to use, and the next-hop router IP for packet forwarding. to determine which interface and router to use.

  • Entries list destination, interface, and router IP
  • If multiple entries match, the longer prefix (most specific) wins
  • The default gatewayThe router used when no specific routing entry matches. Indicated by the 0.0.0.0/0 entry. (0.0.0.0/0) catches all unmatched traffic

IPv6 Networking

IPv6 uses 128-bit addresses shown as eight colon-separated groups of four hexadecimal digits. RHEL operates in dual-stack modeRunning IPv4 and IPv6 simultaneously on the same interface. Red Hat Enterprise Linux does this by default. by default.

IPv6 Address Shortening

# Full form 2001:0db8:0000:0010:0000:0000:0000:0001 # Omit leading zeros 2001:db8:0:10:0:0:0:1 # Use :: for one consecutive zero run 2001:db8:0:10::1
  • Suppress leading zeros (e.g., 0010 becomes 10)
  • Use :: for one consecutive run of all-zero groups (only once per address)
  • If two equal-length zero runs exist, shorten the leftmost
  • Always use lowercase (a-f)
⚠Important

When including a port after an IPv6 address, use square brackets: [2001:db8:0:10::1]:80

IPv6 Autoconfiguration

Stateless Address Autoconfiguration (SLAAC)

The host auto-configures a fe80::/64 link-local address, sends a router solicitation to ff02::2, and receives a network prefix from the router. The radvd package provides SLAAC on RHEL.

DHCPv6

Uses link-local addresses and multicast (not broadcast). Client sends to ff02::1:2 on UDP port 547. Server replies on UDP port 546. The kea package in RHEL 10 provides DHCPv6.

Manual Configuration

Select interface IDs and configure them manually. Some IDs are reserved (e.g., all-zeros subnet router anycast).

Static entries in /etc/hosts. Simple but requires manual updates on every system.

Validating Network Configuration

The ip command provides comprehensive details about network interfaces, addresses, and routes.

Identifying Network Devices

[user@host ~]$ ip link show 1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 state UNKNOWN link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00 2: ens3: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 state UP link/ether 52:54:00:00:fa:0a brd ff:ff:ff:ff:ff:ff 3: ens4: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 8942 state UP link/ether 52:54:00:01:fa:1e brd ff:ff:ff:ff:ff:ff

Displaying IP Addresses

[user@host ~]$ ip addr show ens3 2: ens3: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 state UP link/ether 52:54:00:00:fa:0a brd ff:ff:ff:ff:ff:ff inet 172.25.250.10/24 brd 172.25.250.255 scope global ens3 inet6 2001:db8:0:1:5054:ff:fe00:b/64 scope global inet6 fe80::5054:ff:fe00:fa0a/64 scope link

Key fields: state UP (active), link/ether (MAC), inet (IPv4), inet6 scope global (routable IPv6), inet6 scope link (link-local).

Tracing Routes

Use tracepath, traceroute, or mtr to trace the path through routers. The MTUMaximum Transmission Unit. The largest packet size (in bytes) that can traverse a link without fragmentation. Ethernet typically uses 1500 bytes. and round-trip times are shown for each hop.

[user@host ~]$ tracepath classroom.example.com 1?: [LOCALHOST] pmtu 1500 1: gateway 0.362ms 1: gateway 0.206ms 2: classroom.example.com 0.650ms reached Resume: pmtu 1500 hops 2 back 2

Hands-On Lab: Validate Network Configuration

In this lab, you will practice gathering network information and verifying connectivity.

💻Lab Setup

Log in to your workstation machine as the student user.

Step 1: Identify Network Interfaces

List all network interfaces and their states:

[student@workstation ~]$ ip link show

Identify which interfaces are state UP and note their MAC addresses.

Step 2: View IP Addresses

Display detailed address information:

[student@workstation ~]$ ip addr show

Note the IPv4 address, prefix length, and any IPv6 addresses for each interface.

Step 3: Examine the Routing Table

[student@workstation ~]$ ip route [student@workstation ~]$ ip -6 route

Identify the default gateway and directly connected networks.

Step 4: Test Connectivity

# Ping the default gateway [student@workstation ~]$ ping -c3 172.25.250.254 # Trace the route to an external host [student@workstation ~]$ tracepath classroom.example.com

Step 5: Check Listening Services

[student@workstation ~]$ ss -tlnp

Identify which services are listening and on which ports.

Step 6: Verify Name Resolution

[student@workstation ~]$ cat /etc/resolv.conf [student@workstation ~]$ host classroom.example.com

Chapter Summary

Key Takeaways

  • The TCP/IP model has four layers: Application, Transport, Internet, and Link.
  • Network interface names on RHEL are persistent across reboots. Ethernet uses en, WLAN uses wl, and WWAN uses ww.
  • IPv4 addresses are 32-bit numbers divided into a network prefix and host identifier. Configurable via DHCP or manually.
  • IPv6 addresses are 128-bit hexadecimal numbers supporting SLAAC, DHCPv6, and manual configuration. RHEL uses dual-stack by default.
  • Routing tables determine how packets are forwarded. The default gateway (0.0.0.0/0) handles unmatched traffic.
  • The ip command displays interfaces (ip link), addresses (ip addr), and routes (ip route).
  • Connectivity testing uses ping/ping6, tracepath, and mtr.
  • Socket statistics from ss show listening and established connections (replacing netstat).
