Understanding NetworkManager

In RHEL systems, you use the NetworkManager serviceA daemon that monitors and manages network settings. It provides network and VPN connection status in the GNOME system menu and exposes a D-Bus API. to monitor and manage network settings. The NetworkManager service provides the network and VPN connection status shown in the system menu of the GNOME graphical environment.

Devices and Connections

NetworkManager manages two key concepts:

  • A deviceAny physical or virtual network interface that handles network traffic (e.g., ens3, wlan0). NetworkManager detects devices automatically. is any physical or virtual network interface that handles network traffic.
  • A connectionAlso known as a network profile. Contains configuration settings (IP address, gateway, DNS) for a single network device. Stored as .nmconnection files. (also called a network profile) has related configuration settings for a single network device. Each connection must have a unique name or ID.

Connection configuration files are stored in the /etc/NetworkManager/system-connections/ directory.

Configuring Networking with nmcli

Use the nmcliNetworkManager Command Line Interface. A utility to create, display, edit, delete, activate, and deactivate network connections and to control device status. Accepts abbreviated options. utility to create and edit connection files from the command line. The nmcli command accepts abbreviations: nmcli device disconnect can be nmcli dev dis, and nmcli connection modify can be nmcli con mod.

Viewing Network Information

nmcli device status

Displays the status of all network devices:

[root@host ~]# nmcli dev status DEVICE TYPE STATE CONNECTION ens3 ethernet connected ethernet-ens3 lo loopback connected (externally) lo

The loopback interface is marked external, meaning NetworkManager does not manage that device.

nmcli connection show

[root@host ~]# nmcli con show NAME UUID TYPE DEVICE ethernet-ens3 fab1e257-61a6-3526-abf9-8c50c83a28c7 ethernet ens3 lo 0ef452de-b94d-431f-86f8-f08e85fad57f loopback lo

Use --active to show only active connections.

Connection Details

Lowercase settings (e.g., ipv4.method) are configuration; uppercase (e.g., IP4.ADDRESS) are runtime values.

[root@host ~]# nmcli con show ethernet-ens3 connection.id: ethernet-ens3 connection.autoconnect: yes ipv4.method: auto IP4.ADDRESS[1]: 172.25.250.10/24 IP4.GATEWAY: 172.25.250.254 IP4.DNS[1]: 172.25.250.254

nmcli device show

[root@host ~]# nmcli dev show ens3 GENERAL.DEVICE: ens3 GENERAL.TYPE: ethernet GENERAL.HWADDR: 52:54:00:00:FA:0A GENERAL.STATE: 100 (connected) IP4.ADDRESS[1]: 172.25.250.10/24 IP4.GATEWAY: 172.25.250.254 IP6.ADDRESS[1]: fe80::5054:ff:fe00:fa0a/64

⚠️Important: Disconnect vs. Down

Use nmcli device disconnect to stop traffic on a network interface and deactivate the connection. nmcli con down deactivates a connection, but if autoconnect is enabled the device may immediately reactivate another connection.

4

Adding and Modifying Connections

Adding a New Connection

[root@host ~]# nmcli con add con-name ens4-dhcp type ethernet ifname ens4 Connection 'ens4-dhcp' (1234abcd-...) successfully added.

By default, new connections use ipv4.method auto (DHCP).

[root@host ~]# nmcli con add con-name ens4-static type ethernet ifname ens4 \ ipv4.addresses 192.168.1.100/24 \ ipv4.gateway 192.168.1.1 \ ipv4.dns 8.8.8.8 \ ipv4.method manual Connection 'ens4-static' successfully added.

For static configuration, you must set ipv4.method manual.

[root@host ~]# nmcli con add con-name ens4-ipv6 type ethernet ifname ens4 \ ipv6.addresses 2001:db8:0:1::100/64 \ ipv6.gateway 2001:db8:0:1::1 \ ipv6.method manual

Modifying a Connection

Use nmcli connection modify to change properties. Changes are saved but not applied until reactivated.

[root@host ~]# nmcli con mod ethernet-ens3 ipv4.addresses 172.25.250.20/24 [root@host ~]# nmcli con mod ethernet-ens3 ipv4.gateway 172.25.250.254 [root@host ~]# nmcli con mod ethernet-ens3 ipv4.dns 172.25.250.254 [root@host ~]# nmcli con mod ethernet-ens3 ipv4.method manual [root@host ~]# nmcli con up ethernet-ens3 Connection successfully activated...

Editing Network Configuration Files

Connection profiles use keyfile syntaxAn INI-style configuration format used by NetworkManager. Uses sections in brackets (e.g., [connection], [ipv4]) with key=value pairs. Replaces the deprecated ifcfg format in RHEL 10. stored in /etc/NetworkManager/system-connections/ with the .nmconnection extension.

Example Configuration File

📄 /etc/NetworkManager/system-connections/ethernet-ens3.nmconnection
[connection] id=ethernet-ens3 type=ethernet autoconnect=true interface-name=ens3 [ipv4] method=manual addresses=172.25.250.10/24 gateway=172.25.250.254 dns=172.25.250.254; [ipv6] method=auto

Applying Manual Edits

After editing files manually, run nmcli con reload then nmcli con up.

[root@host ~]# vim /etc/NetworkManager/system-connections/ethernet-ens3.nmconnection [root@host ~]# nmcli con reload [root@host ~]# nmcli con up ethernet-ens3 Connection successfully activated...
⚠️File Permissions Matter

Files must have 0600 permissions (root-only readable). NetworkManager refuses to read files with incorrect permissions.

Configuring Hostnames

A fully qualified domain name (FQDN)A complete domain name specifying the exact location of a host in DNS (e.g., host.example.com). Consists of short hostname plus domain name. is the full name of a host including its domain. The hostnamectlA systemd utility that persistently manages the system hostname. It modifies /etc/hostname and updates the running hostname simultaneously. command is the recommended way to persistently set the hostname.

[root@host ~]# hostnamectl Static hostname: host.lab.example.com Icon name: computer-vm Chassis: vm Operating System: Red Hat Enterprise Linux 10.0 (Coughlan) Kernel: Linux 6.12.0-55.el10.x86_64 [root@host ~]# hostnamectl hostname servera.lab.example.com [root@host ~]# hostname servera.lab.example.com
📚Hostname Types

Static (stored in /etc/hostname, persists across reboots), transient (dynamically assigned by DHCP), and pretty (free-form UTF-8 description for display).

Setting Hostname via nmcli

[root@host ~]# nmcli general hostname servera.lab.example.com

Configuring Name Resolution

A stub resolverA lightweight DNS client on the local machine that forwards queries to a recursive DNS name server. Resolution order is configured in /etc/nsswitch.conf. receives DNS queries from applications and forwards them to a recursive name server. The order is configured in /etc/nsswitch.conf: /etc/hosts is checked first, then DNS.

Static: /etc/hosts

📄 /etc/hosts
127.0.0.1 localhost localhost.localdomain ::1 localhost localhost.localdomain 172.25.254.254 classroom.example.com classroom

Dynamic: /etc/resolv.conf

  • search — Domain names to append for short hostnames
  • nameserver — DNS server IP (up to three for redundancy)
📄 /etc/resolv.conf
# Generated by NetworkManager search example.com nameserver 172.25.250.100
⚠️Do Not Edit resolv.conf Directly

NetworkManager regenerates this file. Set DNS with nmcli con mod instead.

Testing DNS

[root@host ~]# host servera.lab.example.com servera.lab.example.com has address 172.25.250.10 [root@host ~]# getent hosts classroom.example.com 172.25.254.254 classroom.example.com

Chapter Summary

Key Takeaways

  • NetworkManager manages network devices and connections. Files stored in /etc/NetworkManager/system-connections/.
  • nmcli is the CLI tool. nmcli dev status for devices, nmcli con show for connections. Abbreviations accepted.
  • Creating connections: nmcli con add with ipv4.method manual for static, auto for DHCP.
  • Modifying: nmcli con mod saves changes; nmcli con up applies them. Use +/- prefixes.
  • Config files use keyfile format (.nmconnection). After manual edits: nmcli con reload then nmcli con up.
  • nmcli device disconnect reliably deactivates; con down may allow auto-reactivation.
  • hostnamectl persistently sets the hostname via /etc/hostname.
  • Name resolution: /etc/nsswitch.conf controls order; /etc/hosts first, then DNS. Never edit resolv.conf directly.
