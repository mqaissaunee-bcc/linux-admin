About the RHCSA Certification (EX200)

The RHCSA exam (EX200) is the foundational Red Hat certification. It is entirely performance-based — you complete real administrative tasks on a live RHEL 10 system. There are no multiple-choice questions on the real exam.

Red Hat Certified System Administrator: the foundational Red Hat certification validating core Linux sysadmin skills. Prerequisite for RHCE. Earned by passing exam EX200. Performance-based exam: candidates complete real hands-on tasks on a live system rather than answering questions. Every configuration must survive a system reboot to receive credit. Red Hat Enterprise Linux 10: the current major version of RHEL tested on EX200. RHEL is Red Hat's commercial Linux distribution used widely in enterprise and government environments.
🔴This Simulator Mirrors the Real Exam Format

⚠Highest-Failure Domains — Based on Candidate Research

Research from Red Hat learning communities, certified instructors, and exam post-mortems consistently identifies these as the areas where students lose the most points:

  1. SELinux — Many candidates disable it (instant failure). Must stay Enforcing. Tasks 21–24 focus here.
  2. Storage / LVM + fstab — Partition mistakes mid-exam can wipe existing data and cause boot failures. Do storage tasks FIRST. Tasks 25–28.
  3. NFS + autofs + SELinux + Firewall (combined) — Requires coordinating 4 subsystems simultaneously. Tasks 29–30.
  4. Persistence failures — Correct config that does not survive reboot scores zero. Tasks 31–32.
  5. Bash scripting (RHEL 10 new emphasis) — Loops, conditionals, functions, exit code handling. Tasks 33–35.
  6. Local DNF repository — Without a working repo, no packages can be installed, causing cascading failures. Task 40.

Exam Strategy & Critical Commands

The Persistence Rule — Most Common Reason for Lost Points

Every task on EX200 must survive a reboot. After completing all tasks, Red Hat's automated grader reboots the VM and re-checks every requirement. A task that works before the reboot but fails after it receives zero credit.

Task TypeWrong (not persistent)Correct (persistent)
Start a servicesystemctl start httpdsystemctl enable --now httpd
Mount a filesystemmount /dev/sdb1 /dataAdd entry to /etc/fstab, then mount
Open a firewall portfirewall-cmd --add-port=8080/tcpfirewall-cmd --permanent --add-port=8080/tcp && firewall-cmd --reload
Set SELinux booleansetsebool httpd_can_network_connect 1setsebool -P httpd_can_network_connect 1
Change SELinux modesetenforce 0Edit /etc/selinux/config (SELINUX=enforcing)
Add swapswapon /dev/sdb2Add to /etc/fstab, then swapon -a
Hostname changehostname server.example.comhostnamectl set-hostname server.example.com

Never disable SELinux to fix a problem — other tasks will fail if SELinux is not in enforcing mode. Always troubleshoot the context issue.

Study Summary & Next Steps

Key Takeaways

  • EX200 is a 3-hour, live-system performance exam with no multiple choice
  • Passing score is 210/300 (70%); every task must survive a reboot
  • Use systemctl enable --now instead of just start
  • Always add mounts to /etc/fstab; always use firewall-cmd --permanent
  • Never disable SELinux; fix context issues with restorecon and semanage
  • The -P flag on setsebool is required for persistence
