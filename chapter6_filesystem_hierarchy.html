The Linux File-system Hierarchy

The Linux system stores all files on file systemsA file system is a method and data structure that the operating system uses to control how data is stored and retrieved on a storage device., which are organized into a single inverted tree known as a file-system hierarchyThe file-system hierarchy is the organized structure of directories and files on a Linux system, starting from the root directory (/) at the top.. This hierarchy is an inverted tree because the tree root is at the top, and the branches of directories and subdirectories stretch below the root.

ðŸ“ Everything is a File

In Linux, almost everything is represented as a fileâ€”not just documents and images, but also hardware devices, processes, and even system information. This unified approach simplifies system administration and makes Linux remarkably flexible.

The Root Directory

The root directoryThe root directory, denoted by a forward slash (/), is the top-level directory in the Linux file-system hierarchy. All other files and directories exist under the root., denoted by a forward slash (/), is at the top of the file-system hierarchy. The forward slash (/) is also used as a directory separator in file names. For example, if etc is a subdirectory of the / directory, we refer to that directory as /etc. Likewise, if the /etc directory contains a file named issue, we refer to that file as /etc/issue.

ðŸ“ Content Types in Directories

Directory contents are classified by type:

  • Static content remains unchanged until explicitly edited or reconfigured (e.g., binaries, libraries, documentation)
  • Dynamic or variable content might be modified or appended by active processes (e.g., logs, databases)
  • Persistent content remains after a reboot, such as configuration settings
  • Runtime content from a process or from the system is deleted on reboot (e.g., process IDs, temporary files)

Filesystem Hierarchy Standard (FHS)

Red Hat Enterprise Linux uses the Filesystem Hierarchy Standard (FHS)The FHS is a standard that defines the directory structure and directory contents in Unix-like operating systems, ensuring consistency across different Linux distributions. to organize system files. This standard ensures that files are stored in predictable locations, making system administration easier and enabling software to find the files it needs.

Navigation Commands

Three essential commands form the foundation of file-system navigation in Linux. Mastering these commands is crucial for efficient system administration.

pwd - Print Working Directory

The pwd command displays the absolute path of your current working directoryThe current working directory is the directory where you are currently located in the filesystem. It's the context for relative path references.â€”your current location in the filesystem.

pwd Examples student@workstation:~$ pwd /home/student # After changing to /etc student@workstation:/etc$ pwd /etc # The prompt often shows your current directory # but pwd always shows the full absolute path

ðŸ’¡ When to Use pwd

  • When you're unsure of your current location
  • Before running commands that depend on your location
  • In scripts to verify the working directory

cd - Change Directory

The cd command changes your current working directory. It's the primary command for moving through the filesystem.

cd Examples # Change to /etc directory (absolute path) student@workstation:~$ cd /etc student@workstation:/etc$ # Change to a subdirectory (relative path) student@workstation:/etc$ cd sysconfig student@workstation:/etc/sysconfig$ # Go up one directory level student@workstation:/etc/sysconfig$ cd .. student@workstation:/etc$ # Return to your home directory student@workstation:/etc$ cd student@workstation:~$ # Or explicitly use ~ student@workstation:/var$ cd ~ student@workstation:~$ # Return to previous directory student@workstation:~$ cd - /var student@workstation:/var$
ðŸ”„ cd Shortcuts
  • cd or cd ~ â€” Go to your home directory
  • cd - â€” Return to your previous directory
  • cd .. â€” Go up one level (parent directory)
  • cd ~username â€” Go to another user's home directory

ls - List Directory Contents

The ls command lists the contents of a directory. It has many useful options for displaying detailed information.

ls Examples # Basic listing of current directory student@workstation:~$ ls Desktop Documents Downloads Music Pictures Videos # Long format with details student@workstation:~$ ls -l drwxr-xr-x. 2 student student 4096 Jan 10 09:30 Desktop drwxr-xr-x. 2 student student 4096 Jan 10 09:30 Documents ... # Show hidden files (starting with .) student@workstation:~$ ls -a . .. .bash_history .bash_profile .bashrc Desktop ... # Long format + hidden + human-readable sizes student@workstation:~$ ls -lah # List a specific directory student@workstation:~$ ls -l /var/log # List with classification markers student@workstation:~$ ls -F / bin@ boot/ dev/ etc/ home/ lib@ lib64@ ...
Option Description
-lLong format (permissions, owner, size, date)
-aShow all files including hidden (dot files)
-hHuman-readable file sizes (K, M, G)
-RRecursive listing of subdirectories
-tSort by modification time (newest first)
-SSort by file size (largest first)
-FAppend indicator (/ for dirs, @ for links)

Absolute and Relative Paths

A path is the route through the directory tree to locate a file or directory. Understanding the difference between absolute and relative paths is essential for navigating efficiently.

Absolute Paths

An absolute pathAn absolute path is a fully qualified name that specifies the exact location of a file starting from the root directory (/). It always begins with a forward slash. specifies the exact location of a file in the file-system hierarchy. It begins at the root (/) directory and includes each subdirectory that must be traversed to reach the specific file. Every file in a file system has a unique absolute path name.

Absolute Path Examples # All absolute paths start with / /home/student/Documents/report.txt /etc/passwd /var/log/messages /usr/share/doc/bash/README # Using absolute paths with commands student@workstation:~$ cat /etc/hostname workstation.example.com student@workstation:~$ ls -l /var/log/messages -rw-------. 1 root root 245678 Jan 15 12:30 /var/log/messages

Relative Paths

A relative pathA relative path specifies a location starting from your current working directory. It does not begin with a forward slash. identifies a file location relative to the current working directory. Relative paths never start with a forward slash. They use special directory references to navigate:

ðŸ“ . (single dot)

Refers to the current directory. Useful when you need to explicitly reference where you are now.

â¬†ï¸ .. (double dot)

Refers to the parent directory. Use this to move up one level in the directory tree.

ðŸ  ~ (tilde)

Shortcut to your home directory. ~bob refers to user bob's home directory.

Relative Path Examples # Starting from /home/student student@workstation:~$ pwd /home/student # Access Documents subdirectory (relative) student@workstation:~$ cd Documents student@workstation:~/Documents$ pwd /home/student/Documents # Go up one level and into Downloads student@workstation:~/Documents$ cd ../Downloads student@workstation:~/Downloads$ # Copy file from current to parent directory student@workstation:~/Downloads$ cp file.txt .. # Copy file to current directory using . student@workstation:~/Downloads$ cp /etc/hosts . # Navigate up multiple levels student@workstation:/usr/share/doc$ cd ../../.. student@workstation:/$

Hands-on Lab: Navigating the Filesystem

Practice navigating the Linux filesystem using the commands you've learned. Complete each task to build confidence with directory navigation.

Start by exploring your home directory and understanding your current location.

# Verify you're in your home directory student@workstation:~$ pwd /home/student # List all files including hidden ones student@workstation:~$ ls -la # Notice the hidden configuration files starting with . # These include .bashrc, .bash_profile, .bash_history, etc.

Practice moving around the filesystem using absolute paths.

# Navigate to the /etc directory student@workstation:~$ cd /etc student@workstation:/etc$ pwd /etc # List configuration files student@workstation:/etc$ ls # Navigate directly to /var/log student@workstation:/etc$ cd /var/log student@workstation:/var/log$ pwd /var/log # Examine log files student@workstation:/var/log$ ls -l

Practice using relative paths and special directory references.

# Start from /var/log student@workstation:/var/log$ pwd /var/log # Move up one level using .. student@workstation:/var/log$ cd .. student@workstation:/var$ pwd /var # Move up two levels student@workstation:/var$ cd ../.. student@workstation:/$ pwd / # Return home using ~ student@workstation:/$ cd ~ student@workstation:~$ # Navigate to Documents subdirectory student@workstation:~$ cd Documents

Explore important system directories and understand their contents.

# Explore /usr structure student@workstation:~$ ls /usr bin games include lib lib64 libexec local sbin share src # View /boot contents (boot files) student@workstation:~$ ls /boot # Check symbolic links student@workstation:~$ ls -l /bin lrwxrwxrwx. 1 root root 7 Jan 1 00:00 /bin -> usr/bin # Examine device files student@workstation:~$ ls /dev | head -20 # View system hostname file student@workstation:~$ cat /etc/hostname

Learn to quickly toggle between two directories.

Chapter Summary

Key Takeaways

  • The Linux filesystem is organized as an inverted tree starting from the root directory (/), with all files and directories branching below it.
  • The Filesystem Hierarchy Standard (FHS) defines standard directory locations in RHEL, ensuring consistency and predictability.
  • Important directories include: /etc (configuration), /var (variable data), /home (user homes), /usr (installed software), and /tmp (temporary files).
  • Three essential navigation commands: pwd (print working directory), cd (change directory), and ls (list contents).
  • Absolute paths start with / and specify the complete path from root. Relative paths are interpreted from your current location.
  • Special directory references: . (current directory), .. (parent directory), and ~ (home directory) simplify navigation.
