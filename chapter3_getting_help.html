Introduction to Man Pages

The man command is your primary tool for accessing documentation on a Linux system. Man pages are stored in subdirectories of /usr/share/man and are organized by section number.

Basic Usage

To read a man page, simply type man followed by the topic name:

# Read the man page for the ls command student@workstation:~$ man ls # Read the man page for the passwd command student@workstation:~$ man passwd # Read the man page for the man command itself student@workstation:~$ man man

Man pages follow a consistent structure with standard headings. Not all pages include every heading, as some don't apply to all topics.

LS(1) User Commands LS(1)
NAME
ls - list directory contents
SYNOPSIS
ls [OPTION]... [FILE]...
DESCRIPTION
List information about the FILEs (the current directory by default).
OPTIONS
-a, --all do not ignore entries starting with .
SEE ALSO
dir(1), dircolors(1), vdir(1)
Heading Description
NAME Subject name, usually a command or file name, with a brief description
SYNOPSIS Summary of the command syntax
DESCRIPTION In-depth description of the command or topic
OPTIONS Explanation of available command-line options
EXAMPLES Examples of how to use the command
FILES List of files and directories related to the man page
SEE ALSO Related information, typically other man page topics
BUGS Known bugs or limitations in the software
AUTHOR Information about who contributed to the development

Understanding the Synopsis

The SYNOPSIS section uses a specific notation to describe command syntax:

SYNOPSIS ls [OPTION]... [FILE]... # Notation guide: # [OPTION] - Square brackets mean the item is optional # ... - Ellipsis means the item can be repeated # CAPS - Replace with actual value (e.g., a real filename) # | - Separates mutually exclusive choices # {a|b} - Must choose one of the options

Quick Tips for Reading Man Pages

  • The section number in parentheses (e.g., LS(1)) indicates which manual section the page belongs to
  • Bold text usually represents commands or options to type exactly as shown
  • Italic or underlined text represents variables you should replace with actual values
  • The SEE ALSO section is valuable for finding related commands and concepts

Man Page Sections

Man pages are organized into numbered sections based on the type of content. This organization is important because the same topic name can exist in multiple sections with different meanings.

The Nine Standard Sections

1

User Commands

Commands run from the shell by regular users

2

System Calls

Programming functions to request kernel services

3

Library Functions

Functions provided by program libraries

4

Special Files

Device files and special filesystem entries

5

File Formats

Configuration files and data file formats

6

Games

Games and amusements (rarely used)

7

Conventions

Miscellaneous topics and protocols

8

System Admin

Commands for system administration (root)

9

Kernel

Linux kernel internals (non-standard)

Specifying a Section

Some topics exist in multiple sections. For example, passwd exists as both a command (section 1) and a file format (section 5). To access a specific section:

# View the passwd command documentation (section 1) student@workstation:~$ man 1 passwd # View the passwd file format documentation (section 5) student@workstation:~$ man 5 passwd # Alternative syntax with section number student@workstation:~$ man passwd.5 # List all available sections for a topic student@workstation:~$ man -f passwd passwd (1) - update user's authentication tokens passwd (5) - password file
ðŸ’¡ Pro Tip

When you run man topic without specifying a section, man searches sections in a predefined order (typically 1, 8, 2, 3, 4, 5, 6, 7, 9) and displays the first match. This is why man passwd shows the command, not the file format.

Topic Section 1 (Command) Section 5 (File Format)
passwd Change user password Format of /etc/passwd file
crontab Manage cron jobs Crontab file format
hostname Show/set system hostname Hostname resolution config
shadow N/A Format of /etc/shadow file

Searching Within Man Pages

One of the most powerful features of man pages is the ability to search for specific text:

Key Action
/pattern Search forward for pattern
?pattern Search backward for pattern
n Jump to next match
N Jump to previous match
# Example: Search for "recursive" in ls man page student@workstation:~$ man ls # Once in the man page, type: /recursive # The first match will be highlighted # Press 'n' to find next occurrence # Press 'N' to find previous occurrence

Pattern Matching

The search function supports regular expressions for more powerful searches:

# Search for lines starting with a capital letter /^[A-Z] # Search for "color" or "colour" /colou?r # Search for words ending in "file" /file\b

Jumping to Sections

You can quickly jump to specific sections of a man page:

# Jump to OPTIONS section /^OPTIONS # Jump to EXAMPLES section /^EXAMPLES # Jump to SEE ALSO section /^SEE ALSO

Searching for Man Pages

Sometimes you don't know the exact command name but need to find documentation related to a topic. Linux provides several tools for searching through available man pages.

apropos - Search Man Page Descriptions

The apropos command searches the NAME sections of all man pages for a keyword. It's perfect when you know what you want to do but not the command name.

# Find commands related to "password" student@workstation:~$ apropos password chage (1) - change user password expiry information chpasswd (8) - update passwords in batch mode passwd (1) - update user's authentication tokens passwd (5) - password file ... # Find commands related to "partition" student@workstation:~$ apropos partition addpart (8) - inform kernel of new partition fdisk (8) - manipulate disk partition table parted (8) - a partition manipulation program ...

whatis - Display One-Line Descriptions

The whatis command displays brief descriptions of commands. Unlike apropos, it only matches exact command names.

# Get a quick description of ls student@workstation:~$ whatis ls ls (1) - list directory contents # Get descriptions for multiple commands student@workstation:~$ whatis cat head tail cat (1) - concatenate files and print on stdout head (1) - output the first part of files tail (1) - output the last part of files # Show all sections where passwd exists student@workstation:~$ whatis passwd passwd (1) - update user's authentication tokens passwd (5) - password file

man -k - Keyword Search (Same as apropos)

The man -k option performs the same search as apropos. It's useful to know both forms.

# These commands are equivalent: student@workstation:~$ man -k network student@workstation:~$ apropos network # Search for firewall-related commands student@workstation:~$ man -k firewall firewall-cmd (1) - firewalld command line client firewalld (1) - Dynamic Firewall Manager ...

man -f - Find Exact Matches (Same as whatis)

The man -f option performs the same lookup as whatis.

# These commands are equivalent: student@workstation:~$ man -f crontab student@workstation:~$ whatis crontab # Find all man pages for a specific command student@workstation:~$ man -f crontab crontab (1) - maintain crontab files crontab (5) - files used to schedule commands
âš ï¸ Database Update Required

The apropos and whatis commands rely on a database that must be up to date. If you get "nothing appropriate" errors, the administrator may need to run mandb to update the database. On a fresh system, this database is typically updated automatically.

Using Regular Expressions

Both apropos and man -k support regular expressions:

# Find commands starting with "ssh" student@workstation:~$ apropos '^ssh' # Find commands ending in "ctl" (controller) student@workstation:~$ apropos 'ctl$' # Case-insensitive search student@workstation:~$ apropos -i DNS

Limiting Results by Section

# Show only user commands (section 1) student@workstation:~$ apropos -s 1 copy # Show only admin commands (section 8) student@workstation:~$ apropos -s 8 mount

Other Help Resources

While man pages are the primary documentation system, Linux provides several other ways to get help with commands and configuration.

The --help Option

Most commands support a --help option that displays a quick summary of usage and options. This is faster than opening a full man page when you just need a reminder.

# Get quick help for ls command student@workstation:~$ ls --help Usage: ls [OPTION]... [FILE]... List information about the FILEs. -a, --all do not ignore entries starting with . -l use a long listing format -h, --human-readable print sizes in human readable format ... # Some commands use -h instead of --help student@workstation:~$ grep --help student@workstation:~$ date --help
ðŸ’¡ Tip

Pipe --help output to less if it's too long: ls --help | less

GNU Info Documentation

The GNU Info system provides more detailed documentation for many GNU utilities. Use pinfo for a more user-friendly interface.

# View info documentation with pinfo (recommended) student@workstation:~$ pinfo coreutils # View info for a specific command student@workstation:~$ pinfo grep # Traditional info command student@workstation:~$ info ls

The /usr/share/doc Directory

Many software packages install additional documentation in /usr/share/doc. This can include README files, configuration examples, changelogs, and tutorials.

# List documentation directories student@workstation:~$ ls /usr/share/doc bash httpd openssh bind kernel python3 ... # View README for a specific package student@workstation:~$ ls /usr/share/doc/bash/ CHANGES FAQ INTRO README # Read the README file student@workstation:~$ less /usr/share/doc/bash/README

Chapter Summary

Key Takeaways

  • Man pages are the primary documentation system in Linux, accessed with the man command. They provide comprehensive information about commands, configuration files, and system concepts.
  • Man page sections (1-9) organize documentation by type. User commands are in section 1, file formats in section 5, and admin commands in section 8.
  • Navigation keys like Space (page down), / (search), n (next match), and q (quit) help you efficiently find information in man pages.
  • apropos and man -k search man page descriptions by keyword, helping you find relevant commands when you don't know the exact name.
  • whatis and man -f display brief one-line descriptions of commands and show which sections contain documentation for a topic.
  • The --help option provides quick command-line reference without opening a full man page.
  • GNU Info pages (accessed with pinfo) provide detailed, tutorial-style documentation for many GNU utilities.
  • /usr/share/doc contains additional package documentation including README files, examples, and changelogs.
